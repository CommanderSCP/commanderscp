# tools/openapi

Not an npm workspace package — the committed OpenAPI contract plus the vendored tooling that
gates it.

- `openapi.v1.json` — the committed OpenAPI 3.1 doc emitted by `pnpm --filter @scp/server
openapi:emit` (BUILD_AND_TEST.md §3.2). Regenerated by `pnpm gen`; CI's codegen-drift job
  (`.github/workflows/ci.yml`) fails on any diff.
- `check.sh` — the API breaking-change gate (BUILD_AND_TEST.md §4.3, §6 stage 3, §7 "API breaking
  change" row). Runs vendored `oasdiff breaking` between the committed spec at the merge base with
  `main` and the freshly emitted spec in the working tree. Run `pnpm gen` first. Fully offline —
  no network call anywhere in the script.
- `bin/oasdiff-linux-amd64` — vendored `oasdiff` v1.23.0 release binary (Apache-2.0, LICENSE copy
  at `bin/LICENSE-oasdiff`), downloaded from
  `github.com/oasdiff/oasdiff/releases/download/v1.23.0/oasdiff_1.23.0_linux_amd64.tar.gz` and
  verified against the release's published `checksums.txt`
  (`sha256:972b10535c3db4366b9dc3ebc11ca021279af3095267c3cffdca854a3a3c4f89`). Committed so
  `check.sh` runs air-gapped — satisfies BUILD_AND_TEST.md §1's "vendored so the spec gate runs
  air-gapped".

  Only `linux/amd64` is vendored, matching the self-hosted CI runner architecture
  (`homelab-commanderscp-linux-*` in `.github/workflows/ci.yml`) and `tools/ci-image`, which
  bakes this binary in at `/usr/local/bin/oasdiff`. To run `check.sh` on another host
  architecture (e.g. local dev on macOS/arm64), either run it inside a `linux/amd64` container
  (as CI does, via `tools/ci-image`) or download+verify a matching `oasdiff-<os>-<arch>` binary
  from the same GitHub release and point `OASDIFF_BIN` at it — don't commit binaries for
  architectures nothing in CI or prod actually runs on.

## Wiring status (honest note)

`check.sh` exists, is vendored, and is tested (see the m8-hardening-packaging-rc PR description
for the manual breaking/non-breaking test transcript), but as of this change it is **not yet
called from `.github/workflows/ci.yml`'s `codegen-drift` job** — that job still only does `pnpm
gen && git diff --exit-code`. Wiring `check.sh` into that job (plus giving it enough git history —
`fetch-depth: 0` — to resolve a merge base) is scoped as a small follow-up; this change's mandate
was `tools/ci-image/` plus vendoring `tools/openapi/bin/` + `check.sh` itself, not editing other CI
jobs.

## Updating the vendored binary

1. Pick a release ≥ 1.11 from `github.com/oasdiff/oasdiff/releases`.
2. Download `oasdiff_<version>_linux_amd64.tar.gz` and that release's `checksums.txt`.
3. Verify: `sha256sum -c` (or `shasum -a 256 -c`) the tarball against the matching line in
   `checksums.txt`.
4. Extract, replace `bin/oasdiff-linux-amd64` and `bin/LICENSE-oasdiff`, `chmod +x`, update the
   version/checksum noted above and in `tools/ci-image/Dockerfile`'s comment.
